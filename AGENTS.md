## 1. System Summary
- 본 'Follow Me!' 시스템은 복잡한 웹사이트 사용에 어려움을 겪는 사용자들을 위해, 성공적인 사용자들의 경로를 분석하여 개인화된 단계별 가이드를 제공하는 것을 목표로 합니다. 이를 통해 사용자 경험을 향상시키고 이탈률을 줄이고자 합니다.

## 3. Architecture Overview
- **상호작용 흐름:**
  1. **클라이언트 (브라우저 확장 프로그램):** 사용자의 행동(클릭, 입력 등)을 익명으로 수집하여 백엔드 서버로 전송하고, 서버로부터 받은 가이드를 화면에 오버레이로 표시합니다.
  2. **백엔드 서버:** 클라이언트가 보낸 로그를 수신하고 데이터베이스에 저장합니다.
  3. **분석 모듈:** 데이터베이스의 로그를 주기적으로 분석하여 사용자 그룹을 나누고, 성공 및 실패 경로를 식별합니다.
- **주요 데이터 흐름:** 사용자 행동 로그 (클라이언트 -> 서버 -> DB), 분석 결과 (DB -> 분석 모듈 -> DB), 가이드 데이터 (DB -> 서버 -> 클라이언트).
- **모듈 간 의존성:** 클라이언트는 서버 API에 의존합니다. 분석 모듈은 DB에 직접 의존하며 배치(batch) 작업으로 실행됩니다. 서버와 분석 모듈은 DB를 통해 데이터를 교환하므로 직접적인 의존 관계는 없습니다.

## 4. SDLC & Prototype Review
- **역추론된 요구사항:** `main.py`와 `hello_world.py` 파일이 비어 있어, 코드로부터 구체적인 요구사항을 역추론할 수 없습니다. 요구사항은 `PRD.md`와 `SRS.md` 문서에 명시되어 있습니다.
- **프로토타입 특성 및 품질 이슈:**
  - 현재 프로젝트는 실제 구현이 없는 **문서화 및 설계 단계**에 있습니다.
  - 요구사항 및 개발 가이드(`PRD.md`, `SRS.md`, `GEMINI.md`)는 잘 작성되어 있으나, 구현 코드가 부재한 것이 가장 큰 문제입니다.
- **리팩터링 필요 영역:** 리팩터링에 앞서, `SRS.md`와 `GEMINI.md`에 기술된 아키텍처에 따라 각 구성요소(extension, server, analysis)의 기본 골격(scaffolding)을 구현하는 것이 우선적으로 필요합니다.
